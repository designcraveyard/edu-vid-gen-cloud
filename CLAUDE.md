# edu-vid-gen Plugin — Developer Guide

This file helps Claude Code (and QA team members) understand and modify this plugin.

## What This Plugin Does

Generates complete educational MP4 videos from a topic + class level using a **Google Drive-first workflow**. All artifacts (briefs, timelines, images, clips, finals) live in a shared Google Drive folder. The pipeline includes **review gates** where the client can edit the brief (Google Doc) or timeline (Google Sheet) before production continues.

The pipeline:
1. Collect inputs → 2. Write video brief (Google Doc) → **Review Gate** → 2.1. Character sheets → 2.5. Audio-first timeline (Google Sheet) → **Review Gate** → 3. Keyframe images → 4. Video clips (Veo) → 5. Composite → 6. Post-production

The voiceover is the **master clock** — it dictates clip count, roles (AC/TC), durations, and timing.

## Cloud-Native Workflow

All artifacts are uploaded to Google Drive under a per-project folder structure:

- **Brief** is a Google Doc — the client edits it directly, then replies in Claude Code to continue
- **Timeline** and **tracker** are Google Sheets — the client can adjust clip durations, reorder scenes, or flag clips for regeneration
- The pipeline **stops at review gates** after the brief and after the timeline, waiting for client approval
- The client replies in Claude Code (e.g. "looks good" or "change scene 3") to resume the pipeline
- All generated assets (images, audio, clips, final MP4) are synced to the project's Drive folder

## How to Modify This Plugin

### Fixing prompts (most common)
- Image prompt template: `skills/edu-vid-gen/references/prompting.md` → "Image Prompt Building"
- Veo prompt template: `skills/edu-vid-gen/references/prompting.md` → "Veo Prompt Structure"
- Audio tags: `skills/edu-vid-gen/references/audio-tags.md`

### Fixing flow/phase issues
- Main pipeline phases: `skills/edu-vid-gen/SKILL.md`
- Batch pipeline: `skills/batch-gen/SKILL.md`
- Validation criteria: `skills/edu-vid-gen/references/validation.md`

### Adding a new video backend
1. Create `scripts/backends/{name}.py` following the pattern in `veo.py` or `wan.py`
2. Register in `scripts/backends/__init__.py`
3. Update `scripts/generate-video.py` to accept the new `--backend` value
4. Update `skills/edu-vid-gen/references/api-errors.md` with backend-specific errors

### Adding a new skill
1. Create `skills/{skill-name}/SKILL.md` with frontmatter (name, description)
2. The description field controls when Claude Code triggers the skill — make it specific

## API Keys

Keys are loaded from environment variables. Never hardcode keys in skills or scripts.

| Variable | Service | Required for |
|----------|---------|-------------|
| `ELEVENLABS_API_KEY` | ElevenLabs | Voiceover generation |
| `GEMINI_API_KEY` | Google Gemini | Image generation, validation |
| `TOGETHER_API_KEY` | Together AI | Wan 2.7 video backend |
| gcloud ADC | Google Cloud | Veo video generation |
| `credentials.json` | Google OAuth2 | Google Drive/Docs/Sheets |
| `token.json` | Google OAuth2 | Auto-generated by google-auth.mjs |

Run `/setup` to configure these on a new machine.

## Google Workspace Scripts

| Script | Purpose | Auth |
|--------|---------|------|
| `google-auth.mjs` | One-time OAuth setup | OAuth2 |
| `gdrive.mjs` | Drive operations (folders, upload, download) | OAuth2 |
| `gsheets.mjs` | Sheets operations (create, read, write, tracker) | OAuth2 |
| `gdocs.mjs` | Docs operations (create, read, comments) | OAuth2 |
| `sync-to-drive.mjs` | Upload asset to project subfolder | OAuth2 |
| `read-review.mjs` | Read client edits and feedback | OAuth2 |
| `init-project.mjs` | Create full project folder structure + Sheets | OAuth2 |
| `budget-tiers.mjs` | Budget tier config + cost estimation | none |

## Script Path Convention

All skills reference scripts via `__PLUGIN_DIR__/scripts/`. Claude Code resolves `__PLUGIN_DIR__` to the plugin's installation path at runtime.
